# Add PNG rendering

## Parent

- [PRD](../PRD.md)

## What to build

Add PNG rendering through `diagrampilot render <path> --format svg|png --out
<path>`. SVG remains the default render format. PNG should be produced by
rasterizing the current SVG render output so SVG and PNG stay visually aligned.

Keep renderer integration behind the existing adapter seams and avoid adding
D2-specific DiagramSpec semantics. This preserves the future direction toward a
Native Rendering Engine while still shipping a useful alpha PNG path.

## User stories covered

- 15-19

## Acceptance criteria

- [ ] `diagrampilot render <source> --out <path>` keeps rendering SVG by
      default.
- [ ] `diagrampilot render <source> --format svg --out <path>` renders SVG.
- [ ] `diagrampilot render <source> --format png --out <path>` renders PNG.
- [ ] PNG rendering rasterizes the SVG output generated by the current local
      render path.
- [ ] PNG outputs are valid PNG files and are non-empty.
- [ ] Unsupported render formats fail with repairable diagnostics.
- [ ] Invalid sources fail before render side effects.
- [ ] Existing SVG render behavior and provenance remain unchanged.
- [ ] Tests cover default SVG, explicit SVG, PNG, invalid format, invalid
      source, and adapter boundaries.
